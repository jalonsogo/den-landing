Why den

Guardrails on the box,
not on the agent

Lock down a disposable box once, its filesystem and its network, then let any agent run unattended inside it. The blast radius stops at the box.

Full YOLO, safely

Flip on bypass-permissions and stop approving every command. The agent can install anything, run anything, touch any file. It's sealed inside a box it can't escape, so a bad call never reaches your machine.

How it works

From clone to merge
in four steps

Point den at a repo, pick your agent, and get out of its way. It runs sealed inside a box the whole time.

1

Spin up a sandbox

Pick an agent, a base image and a repo. den boots a fresh, disposable box in seconds.

2

Set the guardrails

Lock down the network with a policy and layer in your own tools and credentials. Guardrails on the box, not the agent.

3

Let it run YOLO

Flip on bypass-permissions and walk away. No more yes / no / cancel. The agent works unattended and can't escape.

4

Review & merge

Every change lands on its own branch. Read the full diff, then merge back into your working tree when it's ready.
